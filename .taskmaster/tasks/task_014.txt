# Task ID: 14
# Title: 배포 및 호스팅
# Status: pending
# Dependencies: 12, 13
# Priority: high
# Description: Firebase Hosting 또는 Vercel/Netlify와 같은 저렴한 대안을 사용하여 애플리케이션을 배포합니다.
# Details:
1. 배포를 위한 프로덕션 빌드를 생성합니다 (`npm run build`).
2. Firebase Hosting을 설정하고 초기화합니다.
3. 애플리케이션을 Firebase Hosting에 배포합니다.
4. 사용자 정의 도메인을 설정합니다 (선택 사항).
5. Vercel 또는 Netlify와 같은 다른 호스팅 옵션을 평가하고 비교합니다.
6. GitHub 리포지토리와의 지속적인 통합/지속적인 배포(CI/CD) 파이프라인을 설정합니다.

```sh
# Firebase CLI를 사용한 배포 단계

# 1. Firebase CLI 설치
npm install -g firebase-tools

# 2. Firebase에 로그인
firebase login

# 3. 프로젝트 초기화
# Hosting을 선택하고 기존 Firebase 프로젝트에 연결합니다.
# 공개 디렉토리로 'build'를 지정합니다.
# 싱글 페이지 앱으로 구성합니다.
firebase init hosting

# 4. 프로덕션 빌드 생성
npm run build

# 5. Firebase에 배포
firebase deploy
```
